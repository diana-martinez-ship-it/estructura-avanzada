# Semana 2 — Objetivos y tareas

Plantilla para la Semana 2. Rellena con las actividades, recursos y entregables específicos.

## Resumen

Objetivo de la semana: avanzar en las funcionalidades iniciales, escribir pruebas y documentar resultados.

## Requisitos y configuración

- Revisa el `README.md` raíz y los scripts de inicio.
- Asegura que el entorno creado en la semana 1 sigue funcionando.

## Actividades sugeridas

1. Implementar o mejorar una funcionalidad mínima.
2. Añadir pruebas unitarias o de integración para lo implementado.
3. Documentar cómo probar localmente y qué endpoints o comandos usar.

## Entregables

- Código funcional y probado.
- Capturas o registros de ejecución.
- Pull request con descripción de cambios.

## Checklist

- [ ] Tarea principal completada
- [ ] Tests añadidos y ejecutados
- [ ] Documentación actualizada
